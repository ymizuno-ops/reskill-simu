# requirements.md
# リスキリング年収シミュレーター 要件定義書

---

## 1. プロジェクト概要

### 1.1 目的

リスキリング（再教育）や異業種転職を検討しているユーザーが、
「未経験で転職したら本当に生活できるのか」という不安を、
公的統計データと機械学習モデルに基づく客観的な数値で解消できるWebアプリを提供する。

### 1.2 ターゲットユーザー

- 転職・キャリアチェンジを検討している社会人（20〜55歳）
- リスキリング投資の費用対効果を事前に確認したい人
- 「平均年収」だけでなく自分の年齢・実績に即した予測を求めている人

### 1.3 解決する課題

| 課題 | 解決策 |
|------|--------|
| 求人サイトの平均年収は自分に当てはまらない | 年齢・勤続年数・実績年収を入力してパーソナライズ |
| 未経験転職の初年度年収が読めない | 「1段下の年齢階級」ベースという統計的根拠で算出 |
| 転職判断に客観的な根拠がない | 50年間の収支シミュレーション・ROIを定量化 |
| 単一モデルの予測に不安がある | 9種類のMLモデルで複数シナリオを比較 |

## 2. 機能要件

### 2.1 入力パラメータ

#### プロフィール設定
| パラメータ | 型 | 範囲 | デフォルト | 説明 |
|---|---|---|---|---|
| 現在の年齢 | int | 20〜65歳 | 30 | ユーザーの現在年齢 |
| 現在の勤続年数 | int | 0〜40年 | 5 | 現職での勤続年数 |
| 現在の年収 | int | 100〜3000万円 | 450 | 昨年の実績年収（万円） |

#### 予測モデル選択
| パラメータ | 型 | 選択肢 |
|---|---|---|
| 使用するAIモデル | select | Ridge / ElasticNet / Custom Ridge / Random Forest / Gradient Boosting / LightGBM / CatBoost / XGBoost / Stacking Ensemble |

#### キャリア選択
| パラメータ | 型 | 説明 |
|---|---|---|
| 現職の大分類 | select | 15カテゴリ（すべて含む） |
| 現職名 | select | 148職種（大分類でフィルタ） |
| 目標の大分類 | select | 15カテゴリ（すべて含む） |
| 目標職種名 | select | 148職種（大分類でフィルタ） |
| 経験引継ぎ率 | int | 0〜100%（スライダー） |

#### 投資設定
| パラメータ | 型 | 範囲 | デフォルト | 説明 |
|---|---|---|---|---|
| 自己投資費用 | int | 0〜500万円 | 50 | リスキリング費用 |

| 将来のCPI | int | 80〜150 | 105 | 物価指数（2020年=100） |

#### リアリティ補正
| パラメータ | 型 | 範囲 | デフォルト | 説明 |
|---|---|---|---|---|
| 転職後の昇給抑制 | int | 0〜50% | 0 | 序盤（〜10年）の昇給を抑制 |
| キャリアリスク係数 | int | 0〜30% | 0 | 10年以降の技術陳腐化リスク |

### 2.2 出力・表示

#### 分析結果（5ブロック）
1. **現状**（現在年齢）: 月収・年収
2. **転職直後**（初年度）: 月収・年収・スキル引継ぎ率の適用状況
3. **転職5年後**: 転職前後の月収・年収・差額
4. **65歳時点での生涯年収**: 転職あり/なしの累積年収・生涯差額（投資コスト控除後）
5. **費用対効果**: 投資回収までの月数・生涯年収ベースのROI

#### グラフ
- **年収推移グラフ**: 現状維持・転職後の年収推移（2軸: 年収 + 累積収支差額）
- **全モデル比較グラフ**: 搭載されている全モデルのサブプロット（動的列数）

#### 補助情報
- **モデル精度情報**: R²スコア・CV R²・MAE（expanderで展開）
- **各モデルの特徴説明**: 9モデルの特徴・使い分けガイド（expanderで展開）
- **スキル引継ぎ率ガイド**: 引継ぎ率別の初年度年収シミュレーション表（expanderで展開）
- **年次詳細テーブル**: 5年刻みの年収推移（現状維持・転職後・年間差・累積差）

### 2.3 シミュレーションロジック要件

#### 転職初年度年収の算出
```
ベース年収     = 転職先職種の「現年齢より1段下の年齢階級」の統計平均年収
初年度年収     = ベース年収 + (同年齢・同勤続の経験者年収 - ベース年収) × スキル引継ぎ率
下限           = ベース年収 × 80%
```

#### スキル引継ぎ率の定義
| 引継ぎ率 | 意味 |
|---|---|
| 0% | 完全未経験（1段下の年齢階級相当） |
| 20% | 前職の汎用スキルが少し評価される |
| 40% | ドメイン知識がある程度活かせる |
| 60% | 近接領域・類似業務からの転職 |
| 80% | かなりのスキルが転用できる |
| 100% | 即戦力（資格・経験が完全移行） |

#### 昇給カーブ
- 2年目以降はMLモデルの予測する昇給カーブに従う
- リアリティ補正パラメータで抑制・リスクを加算
- 65歳以降は年3%逓減

#### マクロ経済調整
```
名目賃金上昇率 = GDP成長率 + (CPI - 100) / 100 × 0.3
```

### 2.4 データパイプライン要件

#### Step1（raw → processed）
- e-stat のxlsx/xlsファイルを解析してCSVに変換
- 旧形式（〜2019年）と新形式（2020年〜）の両フォーマットに対応
- 出力: occupation_wage_all.csv / age_wage_all.csv / experience_wage_all.csv / monthly_labor_all.csv / gdp_annual.csv / cpi_annual.csv

#### Step2（processed → master）
- 職種マスタの作成（148職種・2024年基準）
- 年齢カーブ・経験年数カーブの集計
- ML学習用データセットの構築（5,578サンプル）
- マクロ経済パラメータの算出（10年平均）

#### Step3（訓練）
- 9種類のモデルを訓練・評価・保存（models.pkl）
- 5-fold CVによる汎化性能評価
- Stacking EnsembleはネストCV（計25〜45回の訓練）
- 精度情報をmodel_meta.jsonとして保存

## 3. 非機能要件

### 3.1 パフォーマンス
- アプリ起動時間: models.pkl が存在する場合 5秒以内
- シミュレーション実行時間: ボタン押下から結果表示まで3秒以内
- グラフ描画時間: 各グラフ1秒以内

### 3.2 精度
- 選択モデルの CV R² が 0.75 以上であること（Ridge以上）
- Stacking Ensemble は全ベースモデルを上回ること

### 3.3 UI・UX
- ダークモード完全対応
- 日本語フォントを OS（Windows/macOS/Linux）問わず自動検出・適用
- サイドバーの操作だけでシミュレーション条件をすべて設定できること
- 結果はボタン押下後に表示（セッション状態を保持）

### 3.4 保守性
- データ更新: 毎年 e-stat にデータを追加し `python src/step1〜step3` を再実行すれば更新できること
- モデル追加: `step3_train.py` に訓練関数を追加するだけでアプリに自動反映されること
- Wrapperクラス: pickleで保存できるよう必ずモジュールレベルで定義すること

### 3.5 環境
- Python 3.10 以上
- Streamlit 1.32 以上
- OS: Windows / macOS / Linux（動作確認済み）

---

## 4. 制約・前提条件

- 本アプリは統計的推計であり、個人の実際の収入を保証しない
- データソースは厚生労働省・内閣府・総務省の公的統計に限定
- LightGBM / CatBoost / XGBoost は別途インストールが必要（なくても3モデルで動作）
- .xls 形式ファイルの読み込みには LibreOffice が必要（変換用）
- Stacking Ensemble の訓練には 10〜20 分程度かかる
